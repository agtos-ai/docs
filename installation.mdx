## Requirements

| Component | Version | Required | What it enables |
|-----------|---------|----------|-----------------|
| **Node.js** | 22+ | Yes | Core runtime |
| **Docker** | Latest | No | Convenient Redis and service management |
| **Redis** | 7.2+ with RediSearch | No | Memory, scheduling, devices, user preferences |
| **Ollama** | Latest | No | Local AI routing, intent classification, embeddings |

## Install methods

<Tabs>
  <Tab title="npm (development)">
    ```bash
    git clone https://github.com/agtos-ai/agtos.git
    cd agtos
    npm install
    ```

    Then run `npx agtos setup` for interactive configuration.
  </Tab>
  <Tab title="CLI (global)">
    ```bash
    npm install -g agtos
    agtos setup    # Interactive configuration
    agtos start    # Start the server
    ```
  </Tab>
  <Tab title="Docker">
    ```bash
    git clone https://github.com/agtos-ai/agtos.git
    cd agtos
    docker compose up -d
    ```

    This starts agtOS, Redis, and speaches (STT/TTS fallback server) together. For GPU-accelerated speech processing:

    ```bash
    docker compose -f docker-compose.yml -f docker-compose.gpu.yml up -d
    ```
  </Tab>
  <Tab title="Desktop App">
    Download the native app from [GitHub Releases](https://github.com/agtos-ai/agtos/releases):

    | Platform | Format | Architecture |
    |----------|--------|-------------|
    | macOS | `.dmg` | Apple Silicon + Intel |
    | Linux | `.AppImage`, `.deb` | x86\_64 |
    | Windows | `.msi` | x86\_64 |

    The desktop app bundles the agtOS backend — no Node.js required.
  </Tab>
</Tabs>

## Ports

| Port | Service | Protocol |
|------|---------|----------|
| 3000 | Voice pipeline | WebSocket (`/audio`), WebRTC signaling (`/webrtc`) |
| 4100 | MCP server | HTTP (`POST /mcp`) |
| 4102 | Health, REST API, dashboard, metrics | HTTP |

All ports are configurable via `VOICE_PORT`, `MCP_PORT`, and `HEALTH_PORT`.

## Verify installation

```bash
npx agtos doctor
```

The doctor command runs 12 diagnostic checks and reports pass/warn/fail for each:

- Node.js version, Docker, Redis, Ollama, speech models
- API credentials (validated against actual provider APIs)
- Network connectivity, config file, port availability
- Feature degradation summary

See [Troubleshooting](/guides/troubleshooting) if anything fails.
