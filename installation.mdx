## Requirements

| Component | Version | Required | Enables |
|-----------|---------|----------|---------|
| Node.js | 22+ | **Yes** | Core runtime |
| Docker | Latest | No | Convenient Redis and service management |
| Redis | 7.2+ (with RediSearch) | No | Memory, scheduling, devices, user preferences |
| Ollama | Latest | No | Local AI routing, intent classification, embeddings |
| Claude API key | -- | No | Cloud AI reasoning (pay-per-token or Max subscription) |

## Install Methods

### npm (Development)

```bash
git clone https://github.com/agtos-ai/agtos.git
cd agtos
npm install
```

### CLI (Global)

```bash
npm install -g agtos
agtos setup    # Interactive configuration
agtos start    # Start the server
```

The setup wizard validates API keys against the actual provider API and supports both:
- **API key** (`sk-ant-api03-...`) — pay-per-token from console.anthropic.com
- **Max subscription** (`sk-ant-oat01-...`) — flat-rate Claude subscription

Run `agtos doctor` to check system dependencies and credential status.

### Docker

```bash
docker compose up -d
```

### Desktop App

Download from [GitHub Releases](https://github.com/agtos-ai/agtos/releases).

## Ports

| Port | Service | Protocol |
|------|---------|----------|
| 3000 | Voice pipeline | WebSocket (`/audio`), WebRTC signaling (`/webrtc`) |
| 4100 | MCP server | HTTP (`POST /mcp`) |
| 4102 | Health, REST API, dashboard, Prometheus metrics | HTTP |

All ports are configurable via environment variables: `VOICE_PORT`, `MCP_PORT`, `HEALTH_PORT`.

## Verify Installation

After starting the server, run diagnostics to verify everything is working:

```bash
npx agtos doctor
```

This checks Node.js version, Docker, Redis, Ollama, sherpa-onnx models, API credentials, network connectivity, config file validity, and port availability. Each check reports pass/warn/fail with actionable remediation hints.

See [Troubleshooting](/guides/troubleshooting) for common issues and solutions.
